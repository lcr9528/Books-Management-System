# 飞卢图书与章节导入操作说明

本文说明：**如何从飞卢站点导出书籍数据**（单本书或**排行榜/列表页批量**）、**如何将书目写入本系统**、**如何将章节正文批量写入本系统**。默认以项目根目录 `DRF/` 为例，路径请按本机实际修改。

> **合规提示**：爬虫脚本仅供学习、本地验收与自有数据处理；请遵守飞卢用户协议、著作权法与 robots 约定，勿用于未授权的大规模抓取或传播。

---

## 一、环境准备

1. **Python 虚拟环境**（与后端共用即可）  
   - 后端依赖见 `backend/requirements.txt`（含 Django、PyMySQL 等）。  
   - 爬虫额外需要：`requests`、`beautifulsoup4`（若未装：`pip install requests beautifulsoup4`）。

2. **MySQL 与后端 `.env`**  
   - 复制 `backend/.env.example` 为 `backend/.env`，填写 `MYSQL_DATABASE`、`MYSQL_USER`、`MYSQL_PASSWORD`、`MYSQL_HOST`、`MYSQL_PORT`。  
   - 在 `backend` 目录执行迁移（含章节表）：  
     `python manage.py migrate`  
   - 若启动时报 `MySQLdb` 相关错误：本项目通过 **PyMySQL** 兼容层连接 MySQL，请确认已安装 `PyMySQL` 且 `config/__init__.py` 会在加载配置前读取 `.env`（与仓库当前实现一致）。

3. **工作目录约定**  
   - 爬虫可在**项目根**执行：`python scripts/faloo_novel_spider.py ...`  
   - Django 命令在 **`backend`** 目录执行：`python manage.py ...`（或使用 `..\venv\Scripts\python.exe manage.py ...`）。

## 二、爬取网页书籍（导出到本地目录）

脚本路径：`scripts/faloo_novel_spider.py`  
作用：根据书籍详情页拉取元数据、简介、封面，并按目录抓取章节正文，写入本地目录（默认 `./faloo_export`）。

### 2.1 常用命令

| 场景 | 示例命令 |
|------|-----------|
| 指定书籍详情页 URL | `python scripts/faloo_novel_spider.py --book-url https://b.faloo.com/1525878.html` |
| 指定飞卢书号（数字 ID） | `python scripts/faloo_novel_spider.py --novel-id 1525878` |
| 指定输出根目录 | `python scripts/faloo_novel_spider.py --novel-id 1525878 --out ./faloo_export` |
| 只抓前 N 章（默认 10） | `python scripts/faloo_novel_spider.py --novel-id 1525878 --chapters 20` |
| 合并目录分页后尽量抓全本 | `python scripts/faloo_novel_spider.py --novel-id 1525878 --chapters 0` |
| 请求间隔（秒，默认 0.6） | `python scripts/faloo_novel_spider.py --novel-id 1525878 --delay 1.0` |
| 目录「下一页」最多跟 M 次（默认 100） | `python scripts/faloo_novel_spider.py --novel-id 1525878 --max-toc-pages 50` |

查看全部参数：

```bash
python scripts/faloo_novel_spider.py --help
```

### 2.2 导出目录结构（成功时）

在 `--out` 下会生成（书号为示例）：

```text
faloo_export/
└── novel_{书号}/
    ├── meta.json          # 元数据与章节文件列表（导入书目、章节时优先依赖）
    ├── 简介.txt
    ├── cover.jpg（或其它扩展名）
    └── chapters/
        ├── 0001_第一章.txt
        └── …
```

**说明：**

- **`meta.json`** 由脚本结束时写入；若爬取中断，可能出现**只有 `chapters/` 没有 `meta.json`** 的情况，此时书目批量导入会忽略该目录；章节导入仍可按文件名规则尝试（见下文「仅有章节目录」）。
- 模块顶部的用法示例若含 Windows 路径，脚本内已使用 **`r"""…"""`** 原始字符串，避免 `\U` 等转义错误；你在命令行里传路径时，PowerShell 可用正斜杠或引号包裹路径。

### 2.3 常见问题（爬取阶段）

- **目录解析不到章节**：可能需登录、页面改版或频率限制；可适当增大 `--delay`，或检查站点是否对未登录用户隐藏目录。  
- **`--chapters 0` 很慢**：会跟随多页目录并逐章请求，请合理使用 `--max-toc-pages` 与 `--delay`。

### 2.4 按排行榜 / 列表页批量爬取（多本书）

不必一本本指定书号：把浏览器里打开的 **分类榜、排行榜列表页** URL 传给 **`--ranking-url`**，脚本会：

1. 从列表 HTML 中收集指向 **`https://b.faloo.com/{书号}.html`** 的链接（排除 **`书号_章节号.html`** 这类章节链），去重后按页面顺序排队；  
2. **翻页**：优先跟随页面上的「下一页」；若没有，则尝试把 URL 末尾 **`…_N.html`** 中的 **`N`** 逐页加 1（与常见 `y_*_*_…_1.html` 分页格式一致）；  
3. 对解析到的每一本书依次执行与单本相同的流程（简介、封面、`meta.json`、章节 TXT）。

**与单本互斥**：使用 **`--ranking-url`** 时，不要同时使用 **`--book-url`** 或 **`--novel-id`**。

| 参数 | 说明 |
|------|------|
| `--ranking-url` | 榜单/列表页完整地址（可复制浏览器地址栏） |
| `--max-ranking-pages` | 榜单最多翻 **P** 页（默认 `10`），防止无限翻页 |
| `--max-books` | 解析出很多本书时，只爬前 **K** 本；**`0`** 表示不限制（默认 `0`） |
| 其余 `--chapters`、`--delay`、`--out`、`--max-toc-pages` | 对榜单中的**每一本**书生效，含义与单本模式相同 |

**示例（PowerShell，项目根目录）：**

```powershell
# 从青春校园 + 总点击排序的第 1 页开始，最多翻 5 页榜单，只爬其中前 3 本书，每书前 10 章，间隔1.2s
..\venv\Scripts\Activate.ps1
python scripts/faloo_novel_spider.py --ranking-url "https://b.faloo.com/y_1.html" --out ./faloo_export --chapters 10 --max-ranking-pages 5 --max-books 3 --delay 1.2

```

批量时请务必 **加大 `--delay`**、控制 **`--max-books`** 与 **`--max-ranking-pages`**，避免对站点造成过高压力；若列表需登录或页面改版，可能解析不到链接，需自行调整访问方式（如 Cookie）或等待脚本跟进选择器。

## 三、将书籍导入系统（书目 + 封面 + 简介）

命令：`import_faloo_export`（Django 管理命令）  
作用：在库中为每本飞卢书创建一条 **`Book`**，ISBN 规则为 **`FL` + 飞卢书号**（与目录名 `novel_{书号}` 对应），并写入分类、作者、简介、封面等。

### 3.1 前置条件

- MySQL 可用，`.env` 已配置，`migrate` 已执行。  
- 目标目录下每个 `novel_*` 需具备 **`meta.json`**（见 2.2）。

### 3.2 命令示例

在 **`backend`** 目录：

```bash
# 仅预览将要导入的书（不写库）
python manage.py import_faloo_export --path ../faloo_export --dry-run

# 批量导入整个导出根目录
python manage.py import_faloo_export --path ../faloo_export

# 只导入某一本书的目录
python manage.py import_faloo_export --path ../faloo_export/novel_1525878

# 指定分类名（不存在会自动创建）
python manage.py import_faloo_export --path ../faloo_export --category 同人小说

# 书目已存在时：更新书名、作者、简介、封面（慎用）
python manage.py import_faloo_export --path ../faloo_export/novel_1525878 --force-update
```

### 3.3 行为摘要

- **已存在相同 ISBN**：默认跳过；使用 **`--force-update`** 则更新元数据与封面。  
- **无 `meta.json` 的 `novel_*`**：不会作为一本书导入；命令会提示「跳过 xxx：缺少 meta.json」。处理办法：对该书重新跑完爬虫生成 `meta.json`，或手动补全后再导入。

## 四、将书籍章节导入系统（`BookChapter`）

命令：`import_faloo_chapters`  
作用：把导出目录中的章节 **`.txt`** 批量写入数据库表 **`BookChapter`**，供前端图书详情页「目录」与在线阅读使用。

### 4.1 前置条件

- 已执行包含 **`BookChapter`** 的迁移（如 `books.0003_bookchapter`）：`python manage.py migrate`。  
- 对应 **`Book` 已存在**（ISBN = `FL` + 书号），通常需**先完成第三节「书目导入」**。

### 4.2 命令示例

在 **`backend`** 目录：

```bash
# 预览计划写入的章节数量（不写库）
python manage.py import_faloo_chapters --path ../faloo_export --dry-run

# 批量导入所有有章节数据的书
python manage.py import_faloo_chapters --path ../faloo_export

# 只导入一本书
python manage.py import_faloo_chapters --path ../faloo_export/novel_1525878

# 该书已有章节时：先删除该书全部旧章节再全量重导（慎用）
python manage.py import_faloo_chapters --path ../faloo_export/novel_1525878 --replace
```

### 4.3 章节来源与顺序

1. **优先**：若存在 **`meta.json`** 且含 **`chapters_fetched`**，按其中路径与 `sort_order` 读取文件。  
2. **否则**：扫描 `novel_{书号}/chapters/*.txt`，按文件名 **`数字前缀_标题.txt`**（如 `0001_第一章.txt`）解析排序与标题。  
3. **仅有章节、无 `meta.json` 的单目录**：可将 `--path` 指向该 `novel_*` 目录；根目录批量扫描时也会包含「无 meta 但有 `chapters/*.txt`」且能解析出书号的目录。

### 4.4 行为摘要

- 某书**已有任意章节**且未加 **`--replace`**：整本书跳过，避免误重复。  
- 缺文件、空正文、重复 `order` 会跳过并计入命令结尾统计。

### 4.5 演示用示例章节（可选）

若仅需验收前端目录/阅读 UI，可使用内置示例正文（与飞卢导出无关）：

```bash
python manage.py seed_demo_book_chapters --isbn FL1525878
```

默认示例绑定 ISBN `PG0002701`（公版书演示）；详见命令内说明。

## 五、推荐端到端流程（示例）

以下在 **项目根 `DRF/`** 下用相对路径说明；Windows PowerShell 同理，注意先 `cd` 到对应目录。

```text
1a. 爬取一本书（示例：书号 1525878，前 20 章，输出到 faloo_export）
    python scripts/faloo_novel_spider.py --novel-id 1525878 --out ./faloo_export --chapters 20

    或按榜单批量（示例见上文「2.4」）：
1b. python scripts/faloo_novel_spider.py --ranking-url https://b.faloo.com/y_7_0_0_0_0_3_1.html --out ./faloo_export --chapters 10 --max-books 20 --max-ranking-pages 5

2. 进入 backend，迁移（首次或模型变更后）
   cd backend
   python manage.py migrate

3. 书目入库
   python manage.py import_faloo_export --path ../faloo_export

4. 章节入库
   python manage.py import_faloo_chapters --path ../faloo_export

5. 启动后端与前端，在图书详情页查看「目录」与章节正文
```
